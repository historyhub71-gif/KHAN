# Privacy Policy

**Institute Management System**
Last Updated: July 2026

## Introduction

Institute Management System is designed to help educational institutions manage admissions, interviews, attendance, fees, student progress, notifications, and user management securely.

We respect user privacy and are committed to protecting personal information.

---

# Information We Collect

Depending on the user's role, the application may collect:

## Personal Information

* Full Name
* Email Address
* Phone Number (if provided)
* User Role
* Student ID
* Teacher ID
* Director ID

---

## Student Information

The system may store:

* Admission Details
* Course Information
* Assigned Teacher
* Assigned Class
* Interview Results
* Student Level
* Attendance Records
* Progress Reports
* Fortnight Reviews
* Fee Information
* Payment History
* Notification History

---

## Staff Information

For teachers, interviewers (ASR), directors, and administrators, the system may store:

* Name
* Email Address
* Assigned Responsibilities
* Attendance Records
* Salary Settings (where applicable)
* Notifications

---

# Authentication

Authentication is securely managed using **Supabase Authentication**.

Passwords are never stored directly by the application.

Authentication is handled using secure industry-standard methods.

---

# Fee Information

The system stores educational fee records including:

* Original Fee
* Agreed Fee
* Monthly Fee Payments
* Remaining Balance
* Payment Method
* Payment Date
* Receipt Information

The application does **not** process online payments or store payment card information.

---

# Attendance & Progress

The application stores:

* Student Attendance
* Teacher Attendance
* Interview Assessments
* Student Progress Reports
* Fortnight Review Records
* Academic Recommendations

These records are used only for educational management purposes.

---

# Notifications

The application generates notifications related to:

* Admission Status
* Fee Reminders
* Fee Payments
* Attendance Updates
* Interview Results
* Progress Reviews
* System Announcements

Notification history may be stored for future reference.

---

# Data Storage

Application data is stored using **Supabase (PostgreSQL)**.

Data is protected using authentication, role-based access control, and database security policies.

---

# User Roles

The application provides separate access levels for:

* Administrator
* Director
* Interviewer (ASR)
* Teacher
* Student

Each role can only access information permitted for that role.

---

# Data Sharing

We do not sell, rent, or share personal information with third parties except when required by law or when necessary to operate the application infrastructure.

---

# Data Security

We implement reasonable technical and organizational measures to protect user data, including:

* Authentication
* Role-Based Access Control (RBAC)
* Secure Database Policies
* Encrypted Network Communication (HTTPS)

While we strive to protect all information, no system can guarantee absolute security.

---

# User Responsibilities

Users are responsible for:

* Keeping login credentials secure.
* Using the application responsibly.
* Reporting unauthorized account access immediately.

---

# Data Retention

Educational records may be retained as required by the institution for academic, administrative, and legal purposes.

Institutions using this software are responsible for determining appropriate data retention periods.

---

# Children's Privacy

This application is intended for use by educational institutions.

Student data should only be managed by authorized school personnel or with appropriate authorization from parents or guardians where required.

---

# Changes to This Privacy Policy

This Privacy Policy may be updated periodically.

Changes will become effective once published in the project repository or application.

---

# Contact

For questions regarding this Privacy Policy or the Institute Management System, please contact the project administrator or repository owner.
